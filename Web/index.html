RFirefox

The RFirefox project currently includes both an R package and a Firefox extension. With these two pieces of software, it allows the user to create web pages which contain and can excecute R code. Furthermore, it allows the R engine and the Javascript engine to be aware of and communicate with each other. This will allow statisticians to teach methods of analysis and convey results in rich and interesting new ways.

Authors

The RFirefox project is a collaboration between Duncan Temple Lang and his graduate student, Gabriel (Gabe) Becker. Currently Gabe is the corresponding author/maintainer for the project.

Documentation

Currently, documentation for RFirefox is minimal. This will change over the coming weeks, but for now see the tests and example webpages linked to below (they can also be found in the package itself at RFirefox/inst/tests/*.html and RFirefox/inst/examples/*.hml, respectively). These should and give you a good idea of how to do some of the basic things RFirefox is designed to do.

    Related Packages

    A number of Duncan's solo-authored packages may be relevent to users of RFirefox:

  • RGraphicsDevice
    Create prototypes of new Graphics Devices for R using only R code. The Raphael graphics device which ships with RFirefox was written using this package
  • SpiderMonkey
    The SpiderMonkey package gives (for the most part) a 1-to-1 mapping to the C-level API (JSAPI) for Mozilla's SpiderMonkey javascript engine. NOTE: If you intend to use SpiderMonkey along with the RFirefox R package you should be sure to load SpiderMonkey first, as there are a few functions which need to be masked by RFirefox.
